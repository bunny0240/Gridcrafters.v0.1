FIX 1 — RANKS NOT UPDATING
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Problem: leaderboard rank shown to user is stale, not recalculated
live as XP changes.

The `rank()` window function in the leaderboard VIEW is correct, but
it's only recalculated when the view is queried — it must be re-fetched
after every XP change, not cached.

Fix: in useUserStore (or wherever leaderboard/rank is read), refetch
the user's rank whenever profile.total_xp changes:

  useEffect(() => {
    if (!profile?.id) return
    supabase
      .from('leaderboard')
      .select('rank')
      .eq('id', profile.id)
      .single()
      .then(({ data }) => {
        if (data) dispatch({ type: 'SET_RANK', payload: data.rank })
      })
  }, [profile?.total_xp])

Dashboard stat card "Rank" and Profile page rank badge must both
read from this same store value — never compute rank locally or
cache it beyond this effect.

Also subscribe to realtime on the profiles table broadly (not just
filtered to own id) so rank updates when OTHER users gain XP too:

  const channel = supabase
    .channel('global-xp-changes')
    .on('postgres_changes',
      { event: 'UPDATE', schema: 'public', table: 'profiles' },
      () => {
        // any profile's XP changed — refetch own rank
        supabase.from('leaderboard').select('rank')
          .eq('id', profile.id).single()
          .then(({ data }) => data && dispatch({ type:'SET_RANK', payload:data.rank }))
      })
    .subscribe()

Problem: profiles.level is set once on signup and never recalculates
even though total_xp increases.

Root cause: the level CASE logic inside update_total_xp() trigger
references `total_xp` (old value) instead of the new total — check
and fix the trigger:

  create or replace function public.update_total_xp()
  returns trigger language plpgsql security definer as $$
  declare
    v_new_total integer;
  begin
    v_new_total := (select total_xp from public.profiles where id = new.user_id) + new.amount;

    update public.profiles set
      total_xp = v_new_total,
      level = case
        when v_new_total >= 12000 then 6
        when v_new_total >= 6000  then 5
        when v_new_total >= 3000  then 4
        when v_new_total >= 1500  then 3
        when v_new_total >= 500   then 2
        else 1
      end,
      updated_at = now()
    where id = new.user_id;

    return new;
  end;
  $$;

  async function checkShortcutMasterBadge(userId: string) {
    const { data } = await supabase
      .from('level_completion')
      .select('level_name')
      .eq('user_id', userId)
      .eq('module', 'shortcuts')

    const completed = new Set((data ?? []).map(r => r.level_name.toLowerCase()))
    const allDone = ['rookie','intermediate','advanced']
      .every(lvl => completed.has(lvl))

    if (!allDone) return

    const { data: existing } = await supabase
      .from('user_badges')
      .select('id')
      .eq('user_id', userId)
      .eq('badge_id', 'shortcut_master')
      .maybeSingle()

    if (existing) return // already earned

    await supabase.from('user_badges').insert({
      user_id: userId, badge_id: 'shortcut_master'
    })

    showBadgeUnlockToast('⌨️ Shortcut Master')
  }
